# Dialectica — Parchment Scroll: Implementation Prompt

This document defines how to replace the standard rectangular dialogue blocks with an organic parchment scroll aesthetic. Read entirely before writing any code.

---

## Vision

Every dialogue block (Claim, Steelman, Attack, Socratic, Synthesis) is rendered as a **torn-edge parchment fragment** rather than a rectangle. The overall page feels like a manuscript being written in real time — content appears on aged paper with irregular edges, horizontal guide lines, and warm ink-on-vellum typography.

There are two implementation tiers. Choose based on available development time.

---

## Tier 1 — CSS/SVG Parchment (Recommended for v1)

Pure CSS and inline SVG. No new dependencies. Integrates cleanly into the existing React component tree.

### How it works

Each dialogue block is wrapped in a `ParchmentBlock` component that:
1. Renders an SVG background with an **irregular, hand-torn path** instead of a rectangle
2. Positions the content absolutely over the SVG
3. Adds faint horizontal guide lines across the paper (like ruled parchment)
4. Uses the existing Dialectica color system — warm cream fills, maroon/gold accents

### `ParchmentBlock` component spec

```jsx
<ParchmentBlock
  type="steelman" | "attack" | "socratic" | "synthesis" | "claim"
  height={number}        // estimated px height — SVG scales to fit
  label="STEELMAN"
  animateIn={true}       // fade + translateY on mount
>
  {children}
</ParchmentBlock>
```

### SVG path generation

The irregular edge is generated by a JavaScript function, not hardcoded. This ensures every block looks slightly different:

```js
function parchmentPath(width, height, roughness = 6) {
  // Top edge: slight wave
  const topPts = [];
  for (let x = 0; x <= width; x += width / 12) {
    topPts.push([x, 3 + (Math.random() - 0.5) * roughness]);
  }

  // Bottom edge: more torn, larger variance
  const botPts = [];
  for (let x = width; x >= 0; x -= width / 10) {
    botPts.push([x, height - 4 + (Math.random() - 0.5) * roughness * 1.8]);
  }

  // Left and right edges: subtle undulation
  const leftPts = [];
  for (let y = height; y >= 0; y -= height / 8) {
    leftPts.push([3 + (Math.random() - 0.5) * roughness * 0.6, y]);
  }

  const rightPts = [];
  for (let y = 0; y <= height; y += height / 8) {
    rightPts.push([width - 3 + (Math.random() - 0.5) * roughness * 0.6, y]);
  }

  // Stitch all edges into a smooth path using quadratic curves
  const allPts = [...topPts, ...rightPts, ...botPts, ...leftPts];
  let d = `M ${allPts[0][0]},${allPts[0][1]}`;
  for (let i = 1; i < allPts.length; i++) {
    const mid = [(allPts[i-1][0] + allPts[i][0]) / 2, (allPts[i-1][1] + allPts[i][1]) / 2];
    d += ` Q ${allPts[i-1][0]},${allPts[i-1][1]} ${mid[0]},${mid[1]}`;
  }
  d += ' Z';
  return d;
}
```

**Important:** Generate the path once on component mount using `useMemo` — not on every render. The path must stay stable.

```js
const path = useMemo(() => parchmentPath(width, height, roughness), []);
```

### Horizontal guide lines

Draw 3–5 faint horizontal lines across the SVG interior, spaced ~26px apart, starting ~40px from the top. These simulate ruled parchment:

```jsx
{Array.from({length: 5}).map((_, i) => (
  <line
    key={i}
    x1="16" y1={44 + i * 26}
    x2={width - 16} y2={44 + i * 26}
    stroke="#C4A048" strokeWidth="0.3"
    strokeDasharray="4,14" opacity="0.35"
  />
))}
```

### Content positioning

Use `position: absolute` with generous padding to keep text away from torn edges:

```css
.parchment-content {
  position: absolute;
  top: 0; left: 0; right: 0; bottom: 0;
  padding: 22px 36px 26px 32px;
}
```

### Animation on mount

```css
@keyframes parchment-in {
  from { opacity: 0; transform: translateY(10px); }
  to   { opacity: 1; transform: translateY(0); }
}
.parchment-block { animation: parchment-in 280ms ease-out forwards; }
```

### Attack block exception

The Attack block uses maroon fill with light text. The SVG path should have slightly **more** roughness (`roughness = 10`) to emphasize its adversarial nature. The torn edge on the bottom should be the most jagged of all blocks.

## Tier 2 — Three.js Physics Scroll (Stretch goal for v2)

A full 3D hanging parchment scroll with cloth physics. The entire dialogue thread is rendered onto a single long canvas texture draped over a physically-simulated mesh.

### Architecture

```
Three.js Scene
  ├── ParchmentSystem     — Verlet cloth physics mesh
  ├── ContentManager      — Canvas texture with all dialogue content
  └── InteractionManager  — Raycaster drag/pull interaction
```

### ParchmentSystem

- Grid: 20 columns × dynamic rows (grows as content is added)
- Top row: all particles **fixed** — scroll hangs from a straight horizontal line
- Physics: Verlet integration, 5 constraint resolution iterations per frame
- Gravity: `(0, -9.8, 0)` with `drag = 0.99`
- Material: `MeshStandardMaterial`, `roughness: 0.8`, canvas texture mapped

### ContentManager

Renders all dialogue content onto a single `<canvas>` using the Dialectica design system colors:
- Background: `#F0E1C0` parchment base
- Horizontal grain lines every 2px
- Random noise specks for aged texture
- Font: `Georgia` serif for body, `system-ui` for labels
- Maroon `#6B1020` for headlines and labels
- Gold `#B08B3A` for secondary labels and accents

Converts canvas to `THREE.CanvasTexture` and updates when new content blocks are added.

### InteractionManager

- Raycaster detects pointer intersection with parchment mesh
- On `pointerdown`: find closest non-fixed particle
- On `pointermove`: apply pull force `(dragTarget - particle.position) * 0.5`
- On `pointerup`: release particle, physics resolves naturally
- Result: the scroll sways, bends, and springs back like real paper

### Content streaming integration

When a new LangGraph node completes and new content arrives via SSE:
1. `ContentManager.appendBlock(type, text)` — draws new block onto canvas below existing content
2. `ParchmentSystem.extendRows(n)` — adds `n` new particle rows to bottom of mesh
3. Camera smoothly pans down to reveal new content
4. The scroll physically elongates and hangs deeper

### Tear interaction (optional)

A dashed line appears at the bottom of the current content. If the user drags it downward past a threshold:
1. Detach bottom particle row from constraints
2. Create a separate falling mesh fragment
3. Animate fragment falling with gravity + rotation
4. Scroll body snaps upward slightly (like tearing paper)

## What NOT to do

- Do not use `border-radius` on dialogue blocks — the point is to eliminate rectangular corners entirely
- Do not animate the SVG path itself — the shape is fixed on mount, only opacity and position animate
- Do not use `filter: drop-shadow()` — it conflicts with the torn edge shape and causes aliasing
- Do not regenerate the parchment path on re-renders — `useMemo` is mandatory
- In Tier 2: do not use `OrbitControls` — the camera is fixed, only the scroll moves

## Integration with existing Dialectica components

`ParchmentBlock` is a drop-in wrapper. Replace:

```jsx
<div className="blk steel">
  ...
</div>
```

with:

```jsx
<ParchmentBlock type="steelman" height={180} label="STEELMAN" animateIn>
  ...
</ParchmentBlock>
```

All internal content (`blbl`, `btxt`, `bsrc`, `.ar`, `.rm`) stays exactly as-is. Only the outer container changes shape.
